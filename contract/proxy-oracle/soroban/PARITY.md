# NEAR/Soroban Proxy Oracle Parity

This document pins the behavioral parity baseline between the NEAR and Soroban implementations of the Templar Proxy Oracle.

**Baseline Hash**: `64bf8b821cabbc94e4591ca89997c8ec00f365c7`

## Parity Matrix

| Feature | NEAR Baseline | Soroban Equivalent | Safety Semantic | Known Divergence | Verification Command | Evidence Path |

| SEP-40 Read Surface | `list_ema_prices_no_older_than` | Provided by `Sep40Adapter` contracts, not the runtime. The runtime exposes `aggregated_latest(asset)` + `aggregated_history(asset, records)` in exponent form (`NormalizedPrice { mantissa, expo, timestamp }`); each adapter dispatches its own SEP-40 `base/assets/decimals/resolution/price/prices/lastprice` to the parent's normalized read and rescales by per-adapter `decimals`. SEP-40 metadata (`contractmeta!(key = "sep", val = "40")`) is declared on the adapter, not the runtime. | Fail closed if stale or missing. Per-adapter `decimals` lets feeds publish at different precisions without forcing a single contract-wide value. | NEAR uses custom JSON-RPC for the equivalent batched read. | `cargo test -p templar-proxy-oracle-soroban-sep40-adapter-contract` | `.omo/evidence/task-1-parity-matrix.txt` |

| Governance Lifecycle | `gov_create` / `gov_execute` | `create_proposal` / `execute_proposal` / `cancel_proposal` plus `submit` / `accept` / `revoke` compatibility aliases | Maturity-gated execution by proposal id. No FIFO ordering required. Per-operation TTLs via `OperationKind` / `TtlConfig`. | Soroban `submit`/`accept`/`revoke` delegate to `create_proposal`/`execute_proposal`/`cancel_proposal`. `submit` auto-assigns the next id and uses zero requested TTL. The proposal state machine is now a shared `no_std` `templar-proxy-oracle-governance-kernel` ledger (`next_id`, pending id set, per-operation TTLs) used by both runtimes; proposal bodies are stored per-key (Soroban `Proposal(id)` persistent entries, NEAR `LookupMap`) so each action rewrites only the proposal it touches. | `cargo test -p templar-proxy-oracle-soroban-governance-contract parity -- --nocapture` | `.omo/evidence/task-6-governance-ordering-parity.txt` |
| Role/Authority | `Role` enum + RBAC via `near-sdk-contract-tools::Rbac` | Same `Role` set (`Admin`, `ManualTripper`, `CircuitBreakerOperator`, `ProxyConfigurationManager`) + RBAC via OpenZeppelin `stellar-access`. Last Admin removal rejected. `SetActionTtl` requires `ProxyConfigurationManager`. | Role membership is delegated to an audited RBAC library on each runtime — `near-sdk-contract-tools::Rbac` on NEAR, OpenZeppelin `stellar-access` on Soroban — with identical 4-role semantics, admin override, and last-admin protection. The shared governance kernel performs no role checks; authorization is enforced in each runtime (via `Address::require_auth()` on Soroban, `predecessor_account_id` on NEAR) before the kernel mutates the proposal ledger. `SetRole(account, role, set)` manages all roles through the runtime's RBAC backend. | `cargo test -p templar-proxy-oracle-soroban-governance-contract` | `.omo/evidence/task-1-parity-matrix.txt` |
